ChromePhone OS
Concept Design
A speculative UI/UX exploration of what a Google phone OS might look like if built entirely around Chrome OS principles ‚Äî web-first, Google account‚Äìcentric, and browser-native ‚Äî without Android ever existing.
Philosophy
Everything runs as a web app inside Chrome. Native apps are PWAs (Progressive Web Apps). The browser IS the OS shell.
Shelf / Dock
Borrowed from Chrome OS desktop. Pinned apps live at the bottom, inspired by the Chromebook taskbar rather than Android's dock.
Chrome Tabs on Mobile
Multi-tab browsing is first-class. The tab strip appears in-browser. Switching tabs = switching "apps."
Google Account as Identity
Profile avatar replaces the standard user icon. Cloud sync is always-on. Settings sync across all Chrome devices.
Material You (Adapted)
Google's design language applied to a Chrome-native context: rounded tiles, elevation shadows, the four Chrome brand colors used as system accents.
